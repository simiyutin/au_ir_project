         <!-- Gah, you have to manually restart the app if you change this!--> 
         <div class="magic-block-textarea">
          <p>Apache Ignite provides seamless integrations with Hadoop and Spark. While the Ignite-Hadoop integration allows you to use Ignite File System as a primary caching layer to store HDFS data, Ignite-Spark integration allows you to share state in-memory across multiple spark jobs using an implementation of Spark RDD.</p> 
          <h2 class="header-scroll">
           <div class="anchor waypoint" id="section-ignite-for-spark"></div>Ignite for Spark<a class="fa fa-anchor" href="#section-ignite-for-spark"></a></h2> 
          <p>Apache Ignite provides an implementation of Spark RDD abstraction which allows to easily share state in memory across Spark jobs. The main difference between native Spark RDD and <code>IgniteRDD</code> is that Ignite RDD provides a shared in-memory view on data across different Spark jobs, workers, or applications, while native Spark RDD cannot be seen by other Spark jobs or applications. </p> 
         </div> 
         <div class="magic-block-image"> 
          <figure>
           <a href="https://files.readme.io/yBlxMlVLSCeTmFVlAEtH_spark-ignite-rdd-small.png" class="block-display-image-parent block-display-image-size-smart "><img src="https://files.readme.io/yBlxMlVLSCeTmFVlAEtH_spark-ignite-rdd-small.png"></a> 
          </figure> 
         </div> 
         <div class="magic-block-textarea">
          <p><a href="" ui-sref="docs.show({'doc': 'ignite-for-spark'})" class="doc-link" data-sidebar="ignite-for-spark">Read more</a></p> 
          <h2 class="header-scroll">
           <div class="anchor waypoint" id="section-in-memory-file-system"></div>In-Memory File System<a class="fa fa-anchor" href="#section-in-memory-file-system"></a></h2> 
          <p>One of unique capabilities of Ignite is a distributed in-memory file system called Ignite File System (IGFS). IGFS delivers similar functionality to Hadoop HDFS, but only in memory. In fact, in addition to its own APIs, IGFS implements Hadoop FileSystem API and can be transparently plugged into Hadoop or Spark deployments. </p> 
         </div> 
         <div class="magic-block-image"> 
          <figure>
           <a href="https://files.readme.io/BzYteFQ0RGufwlMclb6z_spark-igfs-small.png" class="block-display-image-parent block-display-image-size-smart "><img src="https://files.readme.io/BzYteFQ0RGufwlMclb6z_spark-igfs-small.png"></a> 
          </figure> 
         </div> 
         <div class="magic-block-textarea">

          <h2 class="header-scroll">
