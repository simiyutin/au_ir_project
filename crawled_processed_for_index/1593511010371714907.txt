          <h3><i title="Info" class="fa fa-info-circle"></i>Apache Ignite is ... </h3> 
          <div class="callout-body">
           <p>the <strong>in-memory</strong> computing platform that is <strong>durable</strong>, <strong>strongly consistent</strong>, and <strong>highly available</strong> with powerful <strong>SQL</strong>, <strong>key-value</strong> and <strong>processing</strong> APIs</p>

         <div class="magic-block-textarea">
          <p>Apache Ignite Integrations documentation contains excessive information on all the integrations existed in between Apache Ignite and other technologies and products. </p> 
          <p>The integrations are intended to simplify coupling of Apache Ignite and other technologies, used in your applications or services, in order to either perform a transition to Apache Ignite smoothly or to boost an existed solution by plugging Ignite into it.</p> 
          <p>The existed integrations are divided into a number of realms covered below. To learn more about Apache Ignite In-Memory Data Fabric, in general, go to the <a href="https://apacheignite.readme.io" target="_self">main documentation site</a>.</p> 
          <h2 class="header-scroll">
           <div class="anchor waypoint" id="section-clustering"></div>Clustering<a class="fa fa-anchor" href="#section-clustering"></a></h2> 
          <p>Apache Ignite can be deployed on-premise or in cloud environments. Ignite cluster can be deployed in literally every well-known cloud environment thanks to its integrations with Amazon AWS, Google Compute Engine and Apache JClouds.</p> 
          <ul> 
           <li><a href="" ui-sref="docs.show({'doc': 'amazon-aws'})" class="doc-link" data-sidebar="amazon-aws">Amazon AWS</a> </li>
           <li><a href="" ui-sref="docs.show({'doc': 'google-compute-engine'})" class="doc-link" data-sidebar="google-compute-engine">Google Compute Engine</a> </li>
           <li><a href="" ui-sref="docs.show({'doc': 'jclouds'})" class="doc-link" data-sidebar="jclouds">JClouds</a> </li>
          </ul> 
          <h2 class="header-scroll">
           <div class="anchor waypoint" id="section-hadoop-and-spark"></div>Hadoop and Spark<a class="fa fa-anchor" href="#section-hadoop-and-spark"></a></h2> 
          <p>Apache Ignite Hadoop Accelerator provides a set of components allowing for in-memory Hadoop job execution and file system operations. As for Spark, Ignite enriches it with an implementation of Spark RDD abstraction which allows to easily share state in memory across Spark jobs.</p> 
          <ul> 
           <li><a href="https://apacheignite-fs.readme.io/docs/hadoop-accelerator" target="_self">Hadoop Accelerator</a> </li>
           <li><a href="https://apacheignite-fs.readme.io/docs/ignite-for-spark" target="_self">Spark Shared RDDs</a> </li>
          </ul> 
          <h2 class="header-scroll">
           <div class="anchor waypoint" id="section-in-memory-caching"></div>In-Memory Caching<a class="fa fa-anchor" href="#section-in-memory-caching"></a></h2> 
          <p>The integrations that fall under this section allows leveraging from Apache Ignite for solely caching purposes. Usually, in such situations, Ignite is enabled at the configuration level and the benefit is that you avoid code level modifications.</p> 
          <ul> 
           <li><a href="" ui-sref="docs.show({'doc': 'hibernate-l2-cache'})" class="doc-link" data-sidebar="hibernate-l2-cache">Hibernate L2 Cache</a> </li>
           <li><a href="" ui-sref="docs.show({'doc': 'mybatis-l2-cache'})" class="doc-link" data-sidebar="mybatis-l2-cache">MyBatis L2 Cache</a> </li>
           <li><a href="" ui-sref="docs.show({'doc': 'spring-caching'})" class="doc-link" data-sidebar="spring-caching">Spring Caching</a> </li>
           <li><a href="" ui-sref="docs.show({'doc': 'web-session-clustering'})" class="doc-link" data-sidebar="web-session-clustering">Web Session Clustering</a> </li>
          </ul> 
          <h2 class="header-scroll">
           <div class="anchor waypoint" id="section-osgi-support"></div>OSGi Support<a class="fa fa-anchor" href="#section-osgi-support"></a></h2> 
          <p>In order to facilitate the deployment of the different Ignite modules, along with their dependencies, Apache Ignite offers a set of Karaf features packaged in a feature repository. This makes it possible to quickly provision Ignite onto the OSGi environment by means of a single command in the Karaf shell.</p> 
          <ul> 
           <li><a href="" ui-sref="docs.show({'doc': 'installation-in-apache-karaf'})" class="doc-link" data-sidebar="installation-in-apache-karaf">Installation in Apache Karaf</a> </li>
          </ul> 
          <h2 class="header-scroll">
           <div class="anchor waypoint" id="section-integration-with-apache-cassandra"></div>Integration With Apache Cassandra<a class="fa fa-anchor" href="#section-integration-with-apache-cassandra"></a></h2> 
          <p>The integration makes it feasible to use Apache Ignite as a distributed in-memory system together with Apache Cassandra as a persistent store. Once the data is pre-loaded from Cassandra into Ignite you're able to execute ANSI-99 SQL Queries and ACID transactions over this data letting Ignite keeping in sync in memory and disk data sets.</p> 
          <ul> 
           <li><a href="" ui-sref="docs.show({'doc': 'ignite-with-apache-cassandra'})" class="doc-link" data-sidebar="ignite-with-apache-cassandra">Ignite With Apache Cassandra</a></li>
          </ul> 
          <h2 class="header-scroll">
           <div class="anchor waypoint" id="section-streaming-integrations"></div>Streaming Integrations<a class="fa fa-anchor" href="#section-streaming-integrations"></a></h2> 
          <p>Apache Ignite has a variety of integrations with well-known streaming products and technologies like Kafka, Camel or JMS which enables to inject streams of data into Ignite easily and efficiently.</p> 
          <ul> 
           <li><a href="" ui-sref="docs.show({'doc': 'kafka-streamer'})" class="doc-link" data-sidebar="kafka-streamer">Kafka Streamer</a></li>

           <li><a href="" ui-sref="docs.show({'doc': 'mqtt-streamer'})" class="doc-link" data-sidebar="mqtt-streamer">MQTT Streamer</a> </li>
