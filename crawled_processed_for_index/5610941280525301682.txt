     <h1 id="Running_Rosbridge">Running Rosbridge</h1> 
     <span class="anchor" id="line-9-1"></span>
     <strong>Description:</strong> This tutorial shows you how to launch a rosbridge server and talk to it.
     <br> 
     <br> 
     <span class="anchor" id="line-10-1"></span>
     <span class="anchor" id="line-11-1"></span>
     <span class="anchor" id="line-12-1"></span>
     <strong>Keywords:</strong> rosbridge, roslibjs, teleoperation, Robot Web Tools
     <br> 
     <br> 
     <span class="anchor" id="line-13-1"></span>
     <span class="anchor" id="line-14-1"></span>
     <span class="anchor" id="line-15-1"></span>
     <strong>Tutorial Level:</strong> BEGINNER
     <br> 
     <br> 
     <span class="anchor" id="line-16-1"></span>
     <span class="anchor" id="line-17-1"></span>
     <span class="anchor" id="line-18-1"></span>
     <p class="line874"> <span class="anchor" id="line-25"></span><span class="anchor" id="line-26"></span></p>
     <p class="line867"></p>
     <div class="table-of-contents">
      <p class="table-of-contents-heading">Contents</p>

     <p class="line874"> <span class="anchor" id="line-27"></span><span class="anchor" id="line-28"></span></p>
     <p class="line867"> </p>
     <h3 id="Installing_Rosbridge">Installing Rosbridge</h3> 
     <span class="anchor" id="line-29"></span>
     <span class="anchor" id="line-30"></span>
     <p class="line862">Rosbridge depends on a basic installation of ROS. Check out the <a href="/ROS/Installation">ROS Installation Guide</a> for installing ROS on your machine. <span class="anchor" id="line-31"></span><span class="anchor" id="line-32"></span></p>
     <p class="line874">After ROS is installed, you can install Rosbridge from a .deb package: <span class="anchor" id="line-33"></span><span class="anchor" id="line-34"></span></p>
     <p class="line867"><span class="anchor" id="line-35"></span><span class="anchor" id="line-36"></span></p>
     <pre><span class="anchor" id="line-1-2"></span>sudo apt-get install ros-&lt;rosdistro&gt;-rosbridge-suite</pre>
     <span class="anchor" id="line-37"></span>
     <span class="anchor" id="line-38"></span>
     <p class="line874">This will install the suite of rosbridge packages needed to get started. <span class="anchor" id="line-39"></span><span class="anchor" id="line-40"></span></p>
     <p class="line867"> </p>
     <h3 id="Running_Rosbridge-1">Running Rosbridge</h3> 
     <span class="anchor" id="line-41"></span>
     <span class="anchor" id="line-42"></span>
     <p class="line874">After installing ROS and rosbridge, you need to make sure your system is aware of the packages. To set up your environment for ROS and rosbridge: <span class="anchor" id="line-43"></span><span class="anchor" id="line-44"></span></p>
     <p class="line867"><span class="anchor" id="line-45"></span><span class="anchor" id="line-46"></span></p>
     <pre><span class="anchor" id="line-1-3"></span>source /opt/ros/&lt;rosdistro&gt;/setup.bash</pre>
     <span class="anchor" id="line-47"></span>
     <span class="anchor" id="line-48"></span>
     <p class="line862">All that's left is to run rosbridge. To launch rosbridge and its packages like rosbridge_server and rosapi, a <a href="/roslaunch">launch</a> file is included in the install. To launch the file, run: <span class="anchor" id="line-49"></span><span class="anchor" id="line-50"></span></p>
     <p class="line867"><span class="anchor" id="line-51"></span><span class="anchor" id="line-52"></span></p>
     <pre><span class="anchor" id="line-1-4"></span>roslaunch rosbridge_server rosbridge_websocket.launch</pre>
     <span class="anchor" id="line-53"></span>
     <span class="anchor" id="line-54"></span>
     <p class="line862">This will run rosbridge and create a <a class="http" href="http://www.html5rocks.com/en/tutorials/websockets/basics/">WebSocket</a> on port 9090 by default. <span class="anchor" id="line-55"></span><span class="anchor" id="line-56"></span></p>
     <p class="line862">You can configure the port by setting the <tt class="backtick">~/port</tt> param in ROS. An example launch file that will run rosbridge on port 8080 would look like: <span class="anchor" id="line-57"></span><span class="anchor" id="line-58"></span></p>
     <p class="line867"><span class="anchor" id="line-59"></span><span class="anchor" id="line-60"></span><span class="anchor" id="line-61"></span><span class="anchor" id="line-62"></span><span class="anchor" id="line-63"></span><span class="anchor" id="line-64"></span></p>
     <pre><span class="anchor" id="line-1-5"></span>&lt;launch&gt;
<span class="anchor" id="line-2-2"></span>  &lt;include file="$(find rosbridge_server)/launch/rosbridge_websocket.launch" &gt; 
<span class="anchor" id="line-3-2"></span>     &lt;arg name="port" value="8080"/&gt;
<span class="anchor" id="line-4-2"></span>  &lt;/include&gt;
<span class="anchor" id="line-5-2"></span>&lt;/launch&gt;</pre>
     <span class="anchor" id="line-65"></span>
     <span class="anchor" id="line-66"></span>
     <p class="line867"> </p>
     <h3 id="Talking_to_Rosbridge">Talking to Rosbridge</h3> 
     <span class="anchor" id="line-67"></span>
     <span class="anchor" id="line-68"></span>
     <p class="line862">Now that rosbridge has been launched and a <a class="nonexistent" href="/WebSocket">WebSocket</a> connection is available, we can create a basic HTML webpage to send and receive calls to rosbridge. <a href="/roslibjs">Roslibjs</a> is a <a class="nonexistent" href="/JavaScript">JavaScript</a> library that handles the communication for you. Check out the <a href="/roslibjs/Tutorials/BasicRosFunctionality">getting started with roslibjs</a> tutorial to create a webpage with roslibjs and rosbridge. <span class="anchor" id="line-69"></span><span class="anchor" id="line-70"></span></p>
     <p class="line867"><span class="anchor" id="line-71"></span></p>
     <p class="line867"><span class="anchor" id="line-72"></span></p>
